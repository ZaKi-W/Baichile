# 项目协作规则

- 每次任务执行都不要进行深度测试，除非用户明确要求深度测试。
- 除非用户主动要求，否则默认不要视觉化对比。
- 当前阶段一切以微信小程序为优先，默认以小程序作为开发、构建和验证目标。
- 对意图明确、低风险的简单改动直接自主执行，不要向用户请求确认或选择方案。
- 框架已经提供原生解决方案时，优先使用框架方案，不要自行重复实现。
- 每次完成用户要求的代码更新时，将 `apps/client/src/config/code-version.ts` 中的 `CODE_VERSION` 自增 1；同一项任务无论修改多少文件只自增一次。该数字显示在“我的”页面底部“关于白吃了”的版本信息区域，用于确认当前运行的是最新代码。
