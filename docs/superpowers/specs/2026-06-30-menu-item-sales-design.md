# 菜品列表销量设计

## 目标

微信小程序店铺详情页的每个菜品卡片显示该菜品的月销量。

## 数据设计

- 在共享 `MenuItem` 接口中增加必填整数 `monthlySales`。
- 在菜单项校验 schema 中要求 `monthlySales` 为非负整数。
- API 种子数据与客户端本地 Mock 使用菜品和店铺索引生成稳定的模拟月销量，确保接口可用和接口失败回退时展示一致。

## 页面展示

- 将菜品卡片当前的“店长推荐 · 现点现做”占位文案替换为 `月售 {{ item.monthlySales }}`。
- 沿用现有 `product-sales` 样式，不调整卡片布局或其他交互。

## 验证

- 先增加一条店铺页源码测试，要求列表绑定 `item.monthlySales`，并确认测试因功能缺失而失败。
- 完成最小实现后运行该测试以及相关类型检查；不执行深度测试或视觉化对比。
