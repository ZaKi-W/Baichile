# 店铺详情页设计稿还原设计

## 目标

在不改变现有业务功能的前提下，将 `apps/client/src/pages/store/index.vue` 完整还原为
`baichile-merchant-detail(2).html` 的页面结构和视觉风格。

本次覆盖顶部商家信息卡、服务信息条、分类侧栏、分组商品列表和底部悬浮购物车。
设计稿中的原生 JavaScript 交互不迁移，现有 uni-app、Pinia 和页面跳转逻辑保持不变。

## 实现方式

在现有店铺详情页内重组模板并改写局部样式，不拆分新的业务组件，不修改 API、数据模型
或全局样式。规格选择和购物车抽屉继续复用 `SkuSheet` 与 `CartSheet`。

这样可以将视觉改动限制在单页内，同时降低对现有功能和其他页面的回归风险。

## 页面结构

### 商家信息卡

- 使用深绿色渐变、荧光绿色装饰和大圆角，匹配设计稿的沉浸式顶部卡片。
- 返回按钮调用 uni-app 页面返回能力。
- 展示现有店名、描述、评分、距离和月售数据。
- 店铺 Logo 使用店名首字作为稳定占位，不引入新图片资源。
- 保留虚拟内容提示，并改为设计稿内嵌通知条样式。
- 不加入关注等没有现有业务能力的可操作控件。

### 服务信息条

展示“到店自取”“极速出餐”“食材现做”等静态服务标签。它们仅用于还原设计稿的信息密度，
不建立新的状态、接口或点击行为。

### 分类菜单

- 根据商品的 `categoryId` 对 `store.menu` 分组。
- 左侧分类名使用稳定映射；未知分类回退为“其他”或基于出现顺序生成通用名称。
- 右侧按分组渲染商品区块。
- 左侧分类项作为结构和视觉导航，不实现设计稿中的滚动定位、吸附联动或激活态切换。
- 若菜单为空，显示轻量空状态，页面其他区域仍正常渲染。

### 商品卡

- 保留商品名称、副标题、基础价格和规格入口。
- 有 `imageUrl` 时使用真实图片；没有图片时使用固定配色和商品名称末字生成视觉占位。
- 有规格的商品按钮显示“选规格”，无规格的商品显示加号，但二者都继续打开现有
  `SkuSheet`，避免改变当前加购流程。
- 不引入销量、原价、售罄等当前数据模型不支持的业务状态。

### 底部购物车

- 将现有购物车条改为设计稿的深色悬浮胶囊样式，并适配底部安全区。
- 商品数量和总价继续来自 `useCartStore`。
- 点击摘要仍调用 `openCart` 打开 `CartSheet`。
- 点击“去结算”仍阻止事件冒泡并进入现有结算页。
- 无可结算商品时保持禁用状态。

## 数据与功能边界

不修改以下内容：

- `catalogService.store` 的调用方式；
- `StoreDetail`、`MenuItem` 等共享类型；
- 购物车 store 的数据结构和计算逻辑；
- 规格选择、购物车抽屉与结算页面；
- 设计稿之外的页面和全局 UI token。

新增计算属性只负责视图适配，包括商品分组、分类名称和展示占位，不写入业务状态。

## 响应式与兼容性

- 使用 uni-app 支持的 `view`、`text`、`image` 和 `button` 元素。
- 页面以移动端为主，尺寸使用 `rpx`，并处理窄屏下分类栏和商品卡的空间。
- 顶部和底部考虑小程序安全区；右上角不添加业务操作，避免与原生胶囊区域冲突。
- 样式保持 `scoped`，避免影响其他页面。
