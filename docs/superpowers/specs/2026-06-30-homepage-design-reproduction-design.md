# 首页设计稿还原设计

## 目标

将 `baichile-homepage(1).html` 的首页视觉与交互迁移到现有 uni-app 小程序，并新增“发现”底部导航入口。首页继续使用现有 `catalogService.home()` 接口返回的分类和店铺数据，不把设计稿中的示例商家硬编码为业务数据。

## 页面结构

首页使用自定义导航栏并处理状态栏安全区，其他既有业务页继续使用原生导航栏。首页从上到下包含：

1. 收货位置与个人中心入口。
2. 可点击进入搜索页的搜索框和扫一扫视觉按钮。
3. 三屏自动轮播 Hero，保留设计稿文案、配色、圆点控制和轻量切换动画。
4. “今天吃什么”分类宫格，数据来自首页接口；不足八项时展示实际数量，多于八项时首页仅展示前八项。
5. “附近推荐”标题、口碑徽标、横向筛选条和店铺列表。
6. 虚拟派送说明。
7. “首页 / 发现 / 订单 / 我的”四入口悬浮式自定义底栏。

新增发现页，采用与首页一致的页面底色、自定义安全区顶部和底栏视觉。发现页承接首页 Hero 的榜单/探索入口，以现有首页接口数据展示可进入的分类与店铺内容，不新增后端接口。

## 数据与组件

- 分类名称、图标键和店铺字段继续取自 `HomeResponse`。
- 设计稿要求但接口未直接提供的卡片装饰色，按列表索引使用稳定的循环色板。
- 金额、配送费、距离、预计送达、评分、销量和标签按现有字段格式化，不伪造业务指标。
- `StoreCard` 调整为设计稿的紧凑双栏卡片，并继续由首页和其他需要的页面复用。
- 图标优先使用现有 `AppIcon` 与 icon registry；设计稿中的食物视觉可按分类或店铺稳定映射为 Emoji，避免引入远程图片依赖。
- 自定义底栏补充“发现”配置，并根据当前页面路径正确显示选中态。

## 交互

- 点击位置触发现有定位逻辑；定位中显示即时状态。
- 点击头像进入“我的”页，点击搜索框进入搜索页。
- Hero 自动轮播，点击圆点手动切换；离开页面时清理定时器。
- Hero 主按钮分别引导至分类浏览、发现页或虚拟派送演示入口。
- 点击分类进入对应分类页；点击店铺进入对应店铺页。
- 筛选条切换选中状态，并在前端对当前接口结果执行对应排序；无法由现有数据可靠表达的筛选项只改变选中态，不伪造过滤结果。
- 底栏使用 `uni.switchTab` 在四个 tab 页面间切换。

## 状态与兼容性

- 加载、失败、空分类和空店铺状态使用新视觉系统中的轻量状态块，不保留旧页面突兀的通用卡片。
- 使用 uni-app 支持的 Vue、CSS 和生命周期能力，兼容 H5 与微信小程序。
- 首页和发现页关闭原生导航栏；页面顶部按 `safe-area-inset-top` 留出空间。
- 保留现有接口、路由和业务流程，不扩展优惠券等设计稿明确排除的功能。

## 验证范围

遵循项目规则，仅执行与本次改动相关的轻量验证：

- 更新或补充首页结构、发现入口和底栏配置的单元测试。
- 运行客户端相关测试。
- 运行客户端类型检查。
- 执行一次微信小程序或 H5 构建验证。

不执行深度测试，不制作视觉对比图。
