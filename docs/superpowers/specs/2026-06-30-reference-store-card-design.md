# 参考图商家卡片设计

## 目标

商家列表卡片按照用户提供的外卖列表参考图重排，形成“左侧大方图、右侧高密度经营与配送信息”的视觉结构。优惠券等营销信息不在本次范围内。

## 布局

- 卡片使用白色背景、较大的纵向留白和轻圆角。
- 左侧保留现有店名首字占位，不引入图片资源；占位尺寸调整为约 `180rpx × 180rpx`，保持正方形。
- 右侧内容占据剩余宽度，所有长文本允许收缩，店名超长时单行省略。

## 信息层级

右侧按以下顺序展示：

1. 大号粗体店名。
2. 灰色月售信息和绿色店铺标签。
3. 配送价格行：左侧显示起送价和配送费或免配送费；右侧显示距离和分钟数。
4. 口碑行：橙色评分在前，后接橙色模拟热度文案。
5. 最多展示少量已有店铺标签，作为弱化的底部标签行。

评分不再放在店名右侧。删除之前偏通用指标卡的分散布局。

## 模拟热度

在 `StoreSummary` 增加结构化字段 `recentViewers: number`，客户端与 API Mock 使用店铺序号生成稳定且不同的非负整数。卡片展示为“最近24小时xxx人看过”。

## 不包含

- 不新增真实或生成的店铺图片。
- 不实现优惠券、神券、排名、专送角标和右上角菜单。
- 不改搜索、筛选或列表排序逻辑。

## 验证

- 测试卡片内容顺序、关键类名、热度文案和店名省略规则。
- 测试 Mock 店铺包含合法且不同的 `recentViewers`。
- 运行相关测试、类型检查和微信小程序构建；不做视觉化对比。
