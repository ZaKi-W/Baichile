# 商家列表经营与配送信息设计

## 目标

商家列表不再展示“虚拟店铺 · 不会真实配送”，改为展示更接近点餐场景的经营、距离、时效、起送、配送费和评分信息。

## 数据模型

在 `StoreSummary` 增加三个结构化字段：

- `monthlySales: number`：月售单量，使用非负整数。
- `distanceKm: number`：距离公里数，使用非负数。
- `rating: number`：店铺评分，范围为 0 至 5。

现有 `virtualDeliveryMinutes`、`minimumOrderCents` 和 `deliveryFeeCents` 分别作为配送时长、起送金额和配送费。Mock 数据根据店铺序号生成稳定且不同的数值，后续真实接口可直接替换数据源。

## 卡片信息层级

- 第一行：店铺名称；右侧突出显示一位小数的评分，例如 `4.8分`。
- 第二行：`月售 826 · 1.6km · 32分钟`。
- 第三行：`¥15起送 · 配送费 ¥2`；当配送费为 0 时显示 `免配送费`。
- 标签行继续保留。
- 删除卡片中的店铺描述，不再出现“虚拟店铺 · 不会真实配送”。

金额格式去掉无意义的 `.00`，存在角分时保留两位小数。距离固定显示一位小数。

## 影响范围

共用 `StoreCard` 的首页、分类页和搜索页自动采用新样式。接口契约、API Mock/服务 Mock 以及客户端 Mock 都必须补齐新字段，避免类型或运行时数据缺失。

## 验证

- 测试字段格式、付费配送与免配送两种文案。
- 测试 Mock 店铺包含合法的月售、距离和评分。
- 运行相关测试及类型检查，不执行深度测试或视觉化对比。
