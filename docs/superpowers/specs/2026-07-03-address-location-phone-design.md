# 收货地址定位、地点搜索与微信手机号设计

## 目标

修复微信小程序新增收货地址页中的定位和地点搜索，并增加用户主动授权获取微信绑定手机号的能力，同时保留手动填写方式。

## 定位与地点搜索

- `uni.getLocation` 成功后立即更新当前坐标和待保存坐标。
- 调用现有后端逆地址解析接口，将当前位置对应的可读地址回填至地址输入框。
- 逆地址解析失败时保留已取得的坐标，并明确提示地图服务暂不可用，不把 GPS 成功误报为定位失败。
- 附近地点和关键词搜索继续通过后端代理腾讯位置服务，地图 Key 不下发到小程序。
- 后端返回的配额、配置或网络错误需要转换成用户可理解的提示；真正返回空数组时才显示“无结果”。
- 搜索请求使用当前定位得到的城市作为区域约束（可用时），提高小区和写字楼结果相关性。

## 微信手机号

- 手机号字段旁增加微信小程序原生 `button open-type="getPhoneNumber"` 授权入口。
- 用户点击并同意后，前端把一次性 `code` 发送给应用后端。
- 后端使用现有微信小程序 App ID、App Secret 获取接口调用凭证，再调用微信手机号接口换取手机号。
- 后端只返回手机号数据，不向客户端暴露 App Secret 或微信接口调用凭证。
- 用户拒绝、微信接口失败或当前平台不是微信小程序时，仍可手动输入手机号。

## 错误处理

- 地图服务配额耗尽时显示“地图服务今日额度已用完，请稍后再试”，而不是“无结果”。
- 未配置地图 Key、后端不可达和定位权限被拒绝分别显示对应提示。
- 微信手机号授权拒绝不作为页面错误；服务端换号失败时给出可重试提示。
- 后端对空 code、微信错误响应和缺失手机号做输入及响应校验。

## 验证范围

- 为地图请求错误解析、定位成功后的状态更新、微信手机号后端交换逻辑增加轻量单元测试。
- 运行相关测试与 TypeScript 检查，不执行深度测试或视觉化对比。
- 所有代码更新完成后将 `apps/client/src/config/code-version.ts` 中的 `CODE_VERSION` 自增一次。

## 外部配置

- 腾讯位置服务 Key 必须有可用的 WebService API 配额。
- 正式版小程序必须把后端部署到 HTTPS 域名，并在微信公众平台配置为合法 request 域名。
- 微信公众平台需要允许当前小程序使用“手机号快速验证”能力；该能力的资格或费用以后台实际显示为准。
